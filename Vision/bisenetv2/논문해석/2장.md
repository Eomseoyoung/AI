>## Bisenet V2
> ### 2. 관련 연구(Related Work)
> 최근 수년간 이미지 의미 분할(Image Semantic Sgmetation) 분야는 큰 발전을 이루었다.
> 본 절에서는 본 연구와 가장 밀접한 세 가지 연구 흐름에 초점을 맞춘다.
>
>(1) 일반 의미 분할 기법, (2) 실시간 의미 분할 기법, (3) 경량 네트워크 아키텍처


>### 2.1 일반 의미 분할(Generic Semantic Segmentation)
>전통적인 의미 분할 기법들은 다음과 같은 방식에 기반해 왔다.
> * 임계값 선택(threshold selection)기법
> * 영역 성장(region growing)기법
> * 슈퍼픽셀(super-pixel)기법
> * 그래프 기반(graph-based)기법
>
>이들 방법은 수작업 특징을 사용하여 문제를 해결하였다.
>
>이후 완전 합성곱 네크워크(FCN)가 등장하면서, 새로운 세대의 이미 분할 알고리즘들이 다양한 벤치마크에서 지속적으로 최고 성능을 갱신하였다.
>
>이러한 방법들은 두로 두 가지 유형의 백본네크워크 기반으로 한다.
>
>### (1) Dilated Backbone Network
>Dilated Backbone은 다운샘플링 연산을 제거하고, 합성곱 필터를 팽창시켜 고해상도 특징 표현을 유지한다.
>이 구조는 단순한 팽창 합성곱을 기반으로 하기 때문에, 다음과 같은 다양한 연구들이 이를 확장하였다.
>
> * DeepLabv3 : Atrous Spatial Pyramid Plloing(ASPP)을 통해 다중 스케일 문맥 정보 포착
> * PSPNet : 피라미드 풀링 모듈을 사용하여 전역 문맥 정보 통합
> * Attention 기반 기법 :
>     * Self-Attention
>     * Spatial Attention
>     * Channel Attention
> 이러한 기법들은 모두 장거리 문맥 정보(long-range context)를 포착하는 데 초점을 맞춘다.
>
>
>### (2) Encoder-Decoder Backbone Network
>Encoder-Decoder 구조는 상향경로와 스킵 연결을 사용하여 디코더 단계에서 고해상도 특징 맵을 복원한다.
>
>대표적인 예시는 다음과 같다.
> * FCN, Hypercloumns : 저수준 특징을 통합하기 위한 스킵 연결 사용
> * U-Net : 대칭 구조 기반 복원
> * SegNet : 풀링 인덱스 저장 방식
> * RefineNet : 다중 경로 정제
> * LRR : 단계적 재구성
> * GCN : 대형 커널 합성곱
> * DFN : 채널 어텐션
> * HRNet : 다중 해상도 분기 유지
>
>이 구조들은 저수준 디테일과  고수준 의미 정보를 동시에 인코딩한다.
>
> #### 한계점
> Dilated Backbone과 Encoder-Decoder 구조 모두 높은 정확도를 달성하지만, 다음과 같은 문제를 가진다.
>* 추론속도가 느림
>* 연산량과 메모리 사용량이 큼
>  따라서 실시간 환경에는 적합하지 않다.
>
>본 연구에서는 공간 디테일과 이미 정보를 분리 처리함으로써, 정확도와 속도 간의 균형을 달성하는 새로운 구조를 제안한다.


>### 2.2 실시간 의미 분할 (Real-time Semantic Segmentation)
>실시간 의미 분할은 빠른 상호작용과 즉각적인 응답이 필요한 실제 응용에서 점점 더 중요해지고 있다.
>
>대표적인 실시간 분할 기법은 다음과 같다.
>* SegNet : 소형 네크워크 + 스킵 연결
>* ENet : 처음부터 경량 구조로 설계되어 매우 빠른 속도 달성
>* ICNet: 이미지 캐스케이드 구조 사용
>* DLC : 쉬운 영역에 대한 연산 감소
>* ERFNet : 잔차 연결 + 분해 합성곱
>* ESPNet/ESPNetV2 : 효율적인 공간 피라미드 팽창 합성곱
>* GUN : 가이드 업샘플링 모듈
>* DFANet : 특징 재사용을 통한 연산량 감소
>

>### 실시간 기법의 문제점
> 이들 방법은 실시간 속도는 달성하지만, 대부분 다음을 희생한다.
> * 저수준 공간 디테일 손실
> * 정확도 저하
> 즉, 효율성을 위해 정확도를 크게 포기한다는 한계가 있다.
> 본 연구는 저수준 디테일과 고수준 의미 정보를 동시에 고려함으로써 이 문제를 해결하고자 한다.



>### 2.3 경량 아키텍처 (Light-weight Architecture)
>Group Convolution, Depth-wise Convolution, Separavle Convolution의 도입 이후, 경량 네크워크 설계는 급속히 발전하였다.
>
>
>대표적인 경량 모델은 다음과 같다.
>
> * Xception
> * MobileNet
> * ShuffleNet
> 이들 모델은 주료 분류(classfication)과제에서 속도와 정확도의 균형을 성공적으로 달성하였다.
>
>본 연구에서는 단순히 FLOPs만 고려하는 것이 아니라,
>* 연산 복잡도
>* 메모리 접근 비용
>* 실제 추론 속도(real inference spped)
>를 종합적으로 고려하여 실시간 의미 분할에 특화된 경량 네크워크를 설계한다.
